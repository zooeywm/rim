## What you will practice

- opening vertical and horizontal splits
- moving focus between windows
- creating and closing tabs
- switching tabs without losing editor context

<Aside type="tip" title="Mental model">
	In `rim`, windows are splits inside the current tab. Tabs are top-level workspaces that each keep their own active window and buffer order.
</Aside>

## Part 1: create two splits

<Steps>
	1. Start in normal mode.
	2. Press `Space`, then `w`, then `v` to create a vertical split.
	3. Confirm that focus moves into the newly created right-side window.
	4. Press `Space`, then `w`, then `h` to split the active window horizontally.
	5. Confirm that the active tab now contains multiple windows.
</Steps>

## Part 2: move focus between windows

<Steps>
	1. Press `Ctrl+h` to move to the window on the left.
	2. Press `Ctrl+l` to move back to the right.
	3. Press `Ctrl+j` or `Ctrl+k` if the current layout has a lower or upper neighbor.
	4. Repeat until you can predict which pane will receive focus before pressing the key.
</Steps>

## Part 3: create and switch tabs

<Steps>
	1. Press `Space`, then `Tab`, then `n` to create a new tab.
	2. Confirm the status area reports a new tab.
	3. Press `Space`, then `Tab`, then `[` to move to the previous tab.
	4. Press `Space`, then `Tab`, then `]` to move forward again.
	5. Press `Space`, then `Tab`, then `d` to close the current tab when you are done testing.
</Steps>

## Default navigation table

<Tabs syncKey="windows-tabs-nav">
	<TabItem label="Window split">
		Use `<leader>wv` for vertical split and `<leader>wh` for horizontal split.
	</TabItem>
	<TabItem label="Window focus">
		Use `Ctrl+h`, `Ctrl+j`, `Ctrl+k`, and `Ctrl+l` to move focus among visible windows.
	</TabItem>
	<TabItem label="Tab lifecycle">
		Use `<leader><Tab>n` to create a tab and `<leader><Tab>d` to close the current tab.
	</TabItem>
	<TabItem label="Tab switching">
		Use `<leader><Tab>[` and `<leader><Tab>]` to move to the previous or next tab.
	</TabItem>
</Tabs>

## Practical drill

<Steps>
	1. Open one vertical split.
	2. Move focus back and forth with `Ctrl+h` and `Ctrl+l`.
	3. Create a new tab.
	4. Return to the previous tab and confirm the split layout is still there.
	5. Switch back to the new tab and close it.
</Steps>

## Important behaviors

- Closing the current tab does nothing when only one tab exists.
- Window focus movement is constrained to the current tab.
- Buffer navigation and tab navigation are separate concerns; `H` and `L` switch buffers, not tabs.
