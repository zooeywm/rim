## Why this module matters

The key hint window is the fastest way to learn `rim` without leaving the screen. It shows:

- current single-key actions in the active mode
- available continuations for a pending multi-key prefix
- page information when the list is longer than one view

<Aside type="tip" title="Best use">
	Use key hints whenever you remember the first key of a sequence but not the rest. This is especially useful for `<leader>` groups.
</Aside>

## Open the overview

<Steps>
	1. Make sure no other overlay is open.
	2. Press `F1`.
	3. Confirm that a floating hint window appears.
	4. Read the single-key actions for the current mode.
	5. Press `Esc` to close the window.
</Steps>

## Explore a pending prefix

<Steps>
	1. Press `Space` once.  
	   By default, `Space` is the leader key.
	2. Do not complete the sequence yet.
	3. Watch the status bar and hint window update to the pending prefix.
	4. Read the available continuations, such as window and tab groups.
	5. Press `Backspace` to step back one prefix level.
	6. Press `Esc` to cancel the sequence entirely.
</Steps>

## Navigate a long hint list

<Tabs syncKey="key-hints-nav">
	<TabItem label="Line scroll">
		Use `Up` and `Down`, or `Ctrl+p` and `Ctrl+n`, to scroll the hint body one line at a time.
	</TabItem>
	<TabItem label="Page scroll">
		Use `Ctrl+u` and `Ctrl+d` to move by half pages when the hint window spans multiple pages.
	</TabItem>
	<TabItem label="Exit">
		Use `Esc` to close the hint window. Use `Backspace` only when you want to keep the window open and step back one prefix level.
	</TabItem>
</Tabs>

## Practical drill

<Steps>
	1. Press `Space`.
	2. Observe the leader-level choices.
	3. Press `w`.
	4. Observe that the hint window narrows to window-related continuations.
	5. Press `v` to execute vertical split if you want to continue, or `Backspace` twice to walk back safely.
</Steps>

## When the window does not open

- If the command palette is open, its own key handling has priority. Close it first with `Esc`.
- If the workspace file picker is open, `F1` shows the picker-scope keymap instead of the normal-mode overview.
- If you customized `keymaps.toml`, the window will reflect your current runtime bindings after reload.
