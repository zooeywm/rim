## What you will practice

- normal-mode movement
- entering insert mode from different positions
- leaving insert mode cleanly
- one small edit cycle you can repeat on every file

<Aside type="note" title="Starting point">
	Open any file first.
</Aside>

## The default edit loop

<Steps>
	1. **Start in normal mode.**  
	   Look at the status bar. The mode indicator should show `NORMAL`.
	2. **Move before editing.**  
	   Use `h`, `j`, `k`, and `l` to place the cursor exactly where the change should start.
	3. **Enter insert mode.**  
	   Press one of the entry keys below depending on where you want to type.
	4. **Type the change.**  
	   Insert text, use `Backspace` to correct, and `Enter` for a newline.
	5. **Return to normal mode.**  
	   Press `Esc`. The status bar should return to `NORMAL`.
	6. **Verify the result.**  
	   Move once with `h`, `j`, `k`, or `l`. If movement works immediately, you are back in command-oriented navigation.
</Steps>

## Which insert entry should you use

<Tabs syncKey="insert-entry">
	<TabItem label="At cursor">
		Use `i` when you want the next typed character to appear exactly under the current cursor position.
	</TabItem>
	<TabItem label="After cursor">
		Use `a` when you want to append after the current character. `rim` moves one cell right and then enters insert mode.
	</TabItem>
	<TabItem label="New line below">
		Use `o` to create a line below the current line and start typing immediately.
	</TabItem>
	<TabItem label="New line above">
		Use `O` to create a line above the current line and start typing immediately.
	</TabItem>
</Tabs>

## First hands-on exercise

<Steps>
	1. Move to a line you can safely edit.
	2. Press `o`.
	3. Type `temporary note`.
	4. Press `Esc`.
	5. Press `k` to return to the original line.
	6. Press `j` again to confirm the new line is now part of the buffer and navigation still works.
</Steps>

## Insert-mode control keys

- `Esc`: leave insert mode
- `Enter`: insert a newline
- `Backspace`: delete backward
- `Tab`: insert a tab character
- `Left`, `Right`, `Up`, `Down`: move while staying in insert mode

## Common mistakes and the correct interpretation

- If letters like `h` or `j` are inserted into the buffer, you are still in insert mode. Press `Esc` first.
- If `Esc` does not appear to change the text, that is expected. Its job is to change mode, not to edit the buffer.
- If you want help before committing to a longer sequence, press `F1` in the current mode to open key hints.
