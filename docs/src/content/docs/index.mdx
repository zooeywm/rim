## Current focus

- Workspace session persistence is aligned with runtime state, including tabs, windows, buffers, per-window view state, and scratch-buffer history.
- `:qa` is blocked when any buffer is dirty; `:qa!`, `:wqa`, and `:wqa!` are implemented.
- `:wq` now persists workspace session consistently.
- `:yazi` picker integration is stabilized for cursor and window restore behavior.
- Config file changes are hot-reloaded through `FileWatcher` events, with explicit full-table replacement semantics for command and keymap overrides.
